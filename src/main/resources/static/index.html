SQLi feladatok

Feladat 1 — Belépés

Lépj be admin-ként, de anélkül hogy ismernéd az admin jelszavát. A login form tipikus felhasználónév + jelszó ellenőrzést végez, viszont sérülékeny lekérdezéssel.

Várakozás...

Eredmény

Próbálj meg belépni admin-ként.

Segítség és megoldás

Segítség

Az SQL-ben a -- sor-kommentet jelent: az utána jövő rész a szerveren elveszik. Ha a felhasználónév mezőbe olyan szöveget írsz, ami befejezi a query string-literált és kikommentezi a többit, a jelszó-ellenőrzés átugorható.

Próbálj meg úgy gondolni a query-re, mintha te magad írnád: WHERE username = '???' AND password = 'bármi'. Mit írhatnál a ??? helyére, hogy az AND password rész már ne számítson?

Megoldás

Felhasználónév: admin' -- (a -- után egy szóköz kell!)
Jelszó: bármi

A konstruált SQL így néz ki: SELECT id, username, role FROM users WHERE username = 'admin' -- ' AND password = 'bármi'. A -- kikommentezi az AND password ellenőrzést, így a query simán matchel az admin user-re.

Egy másik klasszikus változat: felhasználónév ' OR 1=1 --, ami az első találatot adja vissza (általában a legkisebb ID-jú user-t — ez itt az admin).

Feladat 2 — UNION-based adatkinyerés

Egy nyilvános termékkereső oszlopokat ad vissza. A háttérben azonban van egy másik tábla (users), amiben a felhasználónevek és jelszavak szerepelnek. Lopd ki az admin jelszavát a kereső segítségével.

Segítség és megoldás

Segítség 1 — oszlop-szám

A UNION csak akkor működik, ha a két SELECT ugyanannyi oszlopot ad vissza. Ezt egy ORDER BY n trükkel deríthetjük ki: ha ORDER BY 2 még működik, de ORDER BY 3 hibát dob, akkor a query 2 oszlopos.

Segítség 2 — séma feltárás

H2-ben (és a legtöbb DB-ben) van egy INFORMATION_SCHEMA, amiből megtudhatod a táblákat és oszlopokat. Pl. ' UNION SELECT TABLE_NAME, NULL FROM INFORMATION_SCHEMA.TABLES --.

A users tábla biztosan létezik, és van benne username + password oszlop.

Segítség 3 — típus-kompatibilitás

A UNION-nal egyesített query-knek nem csak oszlopszámban, hanem típusban is illeszkedniük kell. A products.price DOUBLE, a users.password VARCHAR — ezeket nem lehet közvetlenül UNION-olni. Vagy egy NULL-t (ami minden típusra kompatibilis) tegyél a második oszlopba, vagy fűzd össze a username + password szöveget egyetlen oszlopba.

Megoldás

A naív payload (' UNION SELECT username, password FROM users -- ) hibára fut, mert a password VARCHAR, a price DOUBLE — a UNION nem tudja egyesíteni őket.

Egyik bevált változat — a username + password összefűzve az első (VARCHAR) oszlopba, a második oszlopba 0:

' UNION SELECT username || ':' || password, 0 FROM users --

Konstruált query: SELECT name, price FROM products WHERE name LIKE '%' UNION SELECT username || ':' || password, 0 FROM users -- %'

A találati táblában a termékek mellett megjelennek a userek admin:S3cretAdminP@ss formátumban.
